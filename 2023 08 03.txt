데이터베이스는 4가지의 정의를 가져야 한다.(ppt참조)
한조직의 여러 응용 시스템들이 공용으로 사용되기 위해 최소중복으로 통합, 저장된 운영 데이터의 집합

DBMS(Data Base Management System) : 기능 중요(ppt참조)
데이터 추출 : 조회 ==> 별 의미는 없다
데이터 조작(DML) : 데이터 edit
데이터 정의(DDL) : 데이터 구조(table) edit
데이터 제어(DCL) : 보안, 무결성

데이터 중복 최소화, 데이터 무결성 보장(중요) 
무결성 ==> 데이터의 일관성, 정확성

조회하는 창 == ResultSet(조회 결과의 집합)

기본키(Primary Key) : 절대 중복되지 않는 값이 들어가야함
외래키(Foreign Key) : 외부에있는 기본 키, 참조키라고도 한다(외부에있는 다른키를 참조함)
테이블의 한칸한칸을 컬럼값이라고 한다

SQL(질의어) : *중요* ppt참조

DDL : 데이터베이스의 객체(table, 시퀸스)를 정의
DML : 객체 안의 값들을 조작
DCL : 권한 제어
TCL : COMMIT(데이터에 저장된걸 영구 저장), ROLLBACK(데이터에 저장된걸 되돌려줌)

<network와 IO는 아직 깊게 배울 필요 없다. 코드의 흐름 이해와 참고정도만 하자.>
