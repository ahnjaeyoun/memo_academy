Input : 입력
-> 외부에서 내부로 값이 들어오는 것
내부 : 프로그램	외부 : 키보드	값 : 파일		그러므로 입력 : 파일 읽기
Output : 출력
-> 내부에서 외부로 값이 나가는 것
내부 : 프로그램	외부 : 모니터, 콘솔			그러므로 출력 : 파일 쓰기, 내보내기

외부와 내부의 통로를 스트림(Stream)이라고 한다
Stream : 냇물, 시냇물이 흐르다 (단방향)

바이트 기반 스트림 : 문자, 그림, 영상	1byte -> 4byte
문자 기반 스트림 : only문자		2byte -> 4byte int
한글은 2byte

자바는 메모리에 생성된 객체를 파일/네트워크 출력할 수 있다.
객체는 문자가 아니기 때문에 바이트 기반 스트림으로 출력해야한다.
객체를 출력하기 위해서 바이트로 변경(객체의 데이터:필드값을 일렬로 늘어선 연속적인 모양)
해야하는데 이것을 객체 직렬화라고 한다.
반대로, 파일에 저장되어 있거나, 네트워크에서 전송된 객체를 읽을 수도 있는데, 
입력 스트림으로부터 읽어들인 연속적인 바이트를 객체로 복원 -> 역직렬화라고 한다.

네트워크(Network) 그물/일하다 : 통신망
서버 : 제공하는 사람	서비스를 제공하는 프로그램 또는 컴퓨터
클라이언트 : 손님		서비스를 제공 받는 프로그램 또는 컴퓨터

client -> 요청 -> server -> 응답 -> client

IP(Internet Protocol) : 네트워크 상에서 컴퓨터 각각을 식별하기 위해 구분해주는 통신 규약
포트(port)-> 항구에서 배를 대는 항구 : 한 포트당 한 개의 프로그램 (포트번호는 1~65535)(내부적으로 1~1023까지는 이미 사용됨)
소켓(Sochet) : 전구의 밑둥 은색부분 -> 프로세스 간의 통신에 사용되는 연결 부분
TCP : 전달 제어 규약 -> 문제가 있을 시 재전송
UDP : 데이터를 빠르게 보내기만 할 뿐 다른 지원은 없다 -> 실시간 스트리밍에 많이 사용 됨
